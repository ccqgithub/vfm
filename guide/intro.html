# Introduction

Mobx driven form management and validation tool.

# Why?

There is a quite excellent form management and validation tools like react-hooks-form (opens new window) or formik (opens new window), why should I create a new project?

  • Can't use form validations outside of hooks and components. For examples, I wan't to make form validations in a service, witch be used by many components.
  • They all use context level updates, can not make field level or component level updates.

For these reasons, this tool used mobx (opens new window)'s reactivity capability to create form, so we can create and pass form outside of hooks and components, and keep the reactivity of form states.

# Packages

  • vfm: The form management core, use with mobx (opens new window) stand-alone.
  • vfm-react: Use vfm with react.
